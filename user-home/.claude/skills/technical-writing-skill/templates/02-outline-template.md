---

### 2. Demo 演示（如适用）

**章节目标**: {展示最终效果，激发读者兴趣}

**核心内容**:
- 最终效果展示（截图/GIF/视频）
- 核心功能说明
- 技术栈列表

**写作要点**:
- 用视觉化方式展示成果
- 简洁说明 Demo 的功能
- 提供 Demo 链接或 GitHub repo

**预计字数**: 300-500 字

**需要的素材**:
- Demo 截图/视频
- GitHub repo 链接

---

### 3. Overview（概览）

**章节目标**: {为读者建立技术背景和核心概念}

**核心内容**:
- TL;DR（一句话总结）
- 核心概念介绍（2-3 个）
- 技术背景/历史（简述）
- 为什么选择这个技术（与竞品对比，可选）

**写作要点**:
- 用简单的语言解释复杂概念
- 使用类比或比喻（如 "USB-C" = MCP）
- 提供核心概念的 Mermaid 图

**预计字数**: 800-1200 字

**需要的素材**:
- Mermaid 概念图
- 表格（核心概念对比）
- 官方文档链接
