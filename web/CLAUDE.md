# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

This is the frontend of Agentara (web/). See the parent ../CLAUDE.md and ../docs/overview.md for overall project context.

## Commands

```bash
bun dev        # Start dev server on port 8000
bun run build  # Type-check (tsc -b) then build with Vite
bun run lint   # ESLint
```

## Architecture

- Framework: React 19 + TypeScript + Vite 7
- Routing: TanStack Router (file-based, routes in src/routes/). Route tree is auto-generated into src/routeTree.gen.ts — do not edit
manually.
- Styling: Tailwind CSS v4 + Shadcn (new-york style, neutral base color, CSS variables). UI primitives live in src/components/ui/ and
are ESLint-ignored.
- State/Data: TanStack React Query for server state.
- Theme: Dark by default, managed by ThemeProvider in src/components/theme-provider.tsx.

## Path Aliases

- `@/*` → `./src/*`
- `agentara` → `../src/index.ts` (imports shared types from the backend)

## API Proxy

Dev server proxies `/api/*` to `http://localhost:1984/api` (the backend Hono server).

## Conventions

- Import order is enforced by eslint-plugin-import-x: builtins → external → internal (`@/**`) → relative → CSS/assets. Alphabetized, with blank lines between groups.
- Add Shadcn components via `bunx shadcn@latest add <component>`.
- `src/components/ui/` is auto-generated by Shadcn — avoid manual edits.
- Add new components:
  - Use `bunx --bun shadcn@latest add` to add Shadcn components.

## API Layer

- Use Hono's `hc` RPC client with `AppType` (imported from `agentara`) for end-to-end type-safe API calls.
- Wrap every API endpoint in a dedicated React Query hook (`useQuery` / `useMutation`) under `src/api/`.
- Name pattern: `use<Resource><Action>` — e.g. `useSessions`, `useSessionHistory`, `useTasks`, `useTaskDispatch`.
